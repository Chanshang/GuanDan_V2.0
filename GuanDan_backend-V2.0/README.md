# GuanDan Backend V2.0

这是一个用于掼蛋比赛报名、对阵生成、成绩录入和积分统计的后端原型项目。

当前版本已经具备基础业务闭环，但代码仍处于原型阶段，存在以下明显特征：

- 核心逻辑集中在单个 `run.py`
- 页面通过 Flask + Jinja2 `templates` 直接渲染
- 业务规则、数据库访问、接口定义、页面展示耦合较重
- 适合先做业务梳理和模块拆分，再决定是否迁移到 FastAPI

## 当前已实现的核心功能

- 导入 Excel 报名表并写入数据库
- 初始化队伍、队员、等级、轮次数据
- 生成 3 轮对阵
- 限制同办公室不相遇、尽量避免重复等级匹配
- 设置当前轮次
- 展示当前轮次和累计积分
- 录入桌号比分并回写大小分
- 查询指定桌号的比赛信息
- 后台修改指定轮次和桌号的比分

## 当前涉及的数据表

- `team_info`
- `team_level`
- `mini_team_info`
- `office_info`
- `fight_info`

## 建议的开发方向

现阶段先不要把重点放在“换框架”上，而是先完成下面两件事：

1. 把业务逻辑从 `run.py` 中拆出
2. 补齐可重复执行的测试

如果后续目标是：

- 前后端分离
- 对外提供清晰 API
- 自动生成接口文档
- 更方便做接口测试

那么后续迁移到 FastAPI 会更合适。
