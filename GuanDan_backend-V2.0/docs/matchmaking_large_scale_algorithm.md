# generate_round_pairs_large_scale 算法说明

本文梳理 `services/match_algorithm.py` 中 `generate_round_pairs_large_scale(...)` 的设计思路、关键参数含义，以及它与传统算法的关系和复杂度推导。

## 1. 算法是怎么设计的

### 1.1 目标与约束

该函数面向“千人/万人”规模，核心目标是：

1. 每轮每队只打一场（硬约束）。
2. 多轮中尽量减少重复对阵（软约束，优先级最高）。
3. 对手段位分布尽量均匀（软约束，次优先级）。
4. 队伍总数为奇数时，轮空次数尽量均摊（硬约束 + 公平性目标）。

### 1.2 整体结构（按轮次迭代）

算法按 `rounds` 轮迭代，每轮执行以下步骤：

1. **预处理与状态准备**
   - `_build_team_maps(...)`：整理 `team_members`、`team_levels`、`all_teams`，并做输入完整性校验。
   - `_calc_target_exposure(...)`：给每支队伍计算“目标对手段位曝光次数” `target_exposure`。
   - 初始化全局状态：
     - `repeat_count[(a,b)]`：A-B 已交手次数。
     - `exposure[a][rank]`：A 已对阵该段位次数。
     - `bye_count[a]`：A 已轮空次数（奇数队时生效）。

2. **奇数队先选轮空**
   - `_pick_bye_team(...)` 选择 `bye_count` 最小的队伍轮空，保证轮空长期均匀。

3. **构建稀疏候选边**
   - `_build_candidates_for_round(...)` 为每支队伍只保留最多 `candidate_k` 个对手候选，而不是全量枚举所有队伍。
   - 候选优先来自“当前更缺的段位对手池”，并通过 `_edge_cost(...)` 过滤和排序。

4. **求解一轮匹配**
   - `_greedy_match_round(...)` 采用“候选最少优先”的贪心策略，降低死锁概率。
   - 若仍有未匹配队伍，调用 `_repair_unmatched(...)` 做局部交换（2-opt 风格）修复。

5. **分级放宽（保证可落地）**
   - 每轮依次尝试 `relax_level = 0 -> 1 -> 2`：
     - Level 0：不允许重复对阵。
     - Level 1：允许少量重复，但仍重罚重复。
     - Level 2：继续放宽，优先保证全匹配成功。
   - 任一级成功覆盖全队，即接受该轮结果。

6. **更新全局状态**
   - 对本轮每条配对更新 `repeat_count` 与 `exposure`，再进入下一轮。

### 1.3 为什么适合大规模

关键是“**稀疏候选 + 局部修复**”：

- 不做全图最优（避免高复杂度）。
- 每队只看 `candidate_k` 个对手，把单轮成本从近似 `O(N^2)` 压到更接近 `O(N * candidate_k)`。
- 通过分级放宽提高可行性，避免在复杂约束下频繁失败。

## 2. 参数说明：W_repeat / W_quota / W_same_rank / candidate_k

这几个参数共同决定 `_edge_cost(...)` 对“好边/坏边”的判断。

### 2.1 `W_repeat`（重复对阵惩罚权重）

- 作用：惩罚历史已交手的 A-B 边，数值越大越不愿重复。
- 在代码中是“分级权重”：
  - `relax_level=0`：权重大，且直接禁止重复。
  - `relax_level=1/2`：逐步减小，让算法在无解边界时可继续匹配。
- 调参影响：
  - 过小：重复对阵增多。
  - 过大：可能为了避重而牺牲段位均衡，甚至提升局部无解概率。

### 2.2 `W_quota`（段位配额偏差惩罚）

- 作用：惩罚“本次匹配后，某队对某段位曝光超过目标配额”的程度。
- 目标：让每队最终 `exposure` 尽量接近 `target_exposure`。
- 调参影响：
  - 过小：段位均衡效果弱。
  - 过大：可能抬高求解难度，导致更多放宽级别触发。

### 2.3 `W_same_rank`（同段位轻惩罚）

- 作用：当双方段位相同，增加一个轻度惩罚。
- 目标：在不破坏可行性的前提下，鼓励跨段位交手。
- 调参影响：
  - 过小：跨段位促进弱。
  - 过大：同段位对局被过度压制，可能影响可解性和稳定性。

### 2.4 `candidate_k`（每队候选对手上限）

- 作用：控制每支队伍保留多少候选边。
- 它是复杂度和质量的核心平衡杆：
  - 小 `k`：速度快、内存低，但可能更容易局部无解。
  - 大 `k`：质量更高、死锁更少，但时间和内存开销上升。
- 实践建议：
  - 先用中等值（如 20~40）压测。
  - 当队伍规模很大时，优先小步增大 `k`，观察“成功率/耗时”曲线。

## 3. 算法基于什么传统方法？复杂度如何推导？

### 3.1 传统算法来源（混合式）

该实现不是单一经典算法，而是工程化混合方案：

1. **贪心匹配（Greedy Matching）**
   - 每次优先处理候选最少的点，再选代价最低边。
2. **局部搜索/交换修复（Local Search / 2-opt 风格）**
   - 对未匹配残留点，通过替换已有边进行局部重排。
3. **带惩罚的启发式目标函数（Penalty-based Heuristic）**
   - 用加权代价把“减少重复、段位均衡、跨段位倾向”统一到一个评分中。
4. **分级放宽（Lagrangian Relaxation 的工程化近似思路）**
   - 先严格后放宽，先保公平，必要时保可行。

可把它理解为：**“带约束的近似最小代价匹配”的启发式实现**。

### 3.2 时间复杂度（近似）

设：

- `N`：队伍数；
- `R`：轮次 `rounds`；
- `k`：`candidate_k`；
- `I`：修复迭代上限 `max_repair_iter`（通常很小，可视为常数）。

单轮主要成本：

1. 候选构建：每队最多保留 `k` 边，近似 `O(N * k)`。
2. 贪心匹配：每次从候选集中选边，整体近似 `O(N * k)`（实现细节下可有额外常数因子）。
3. 局部修复：`I` 次小规模交换，近似 `O(I * N)` 到 `O(I * N * k)`。

因此单轮近似：

`O(N * k + N * k + I * N * k) = O(N * k)`（当 `I` 为小常数）。

总复杂度近似：

`O(R * N * k)`。

> 这也是源码注释中给出的结论：在稀疏候选前提下，近似线性扩展到大规模。

### 3.3 空间复杂度（近似）

主要状态：

1. `repeat_count`：最坏可能累计到 `O(R * N)` 条已使用边（远小于全图 `O(N^2)`）。
2. `exposure` 与 `target_exposure`：每队 × 段位数，若段位种类为 `L`，则 `O(N * L)`；当前场景 `L=3`，可视为 `O(N)`。
3. 每轮候选边：`O(N * k)`。

因此空间复杂度近似：

`O(N * k + R * N + N * L)`，在固定小 `R/L` 场景下可视作 `O(N * k)`。

---

## 4. 小结（调参优先级建议）

1. 先固定 `candidate_k`，观察成功率和耗时。
2. 重复过多时优先增大 `W_repeat`（或提高严格级别占比）。
3. 段位不均衡时增大 `W_quota`，并检查 `target_exposure` 是否符合预期。
4. 想加强跨段位但不破坏稳定性时，小幅调 `W_same_rank`。

推荐流程：先保“可解 + 稳定”，再迭代公平性指标。
